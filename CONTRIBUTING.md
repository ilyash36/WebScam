## Стандарты кода

### PEP 8

Весь код должен соответствовать стандарту PEP 8:
- Максимальная длина строки: 88 символов
- Используйте trailing commas в многострочных структурах
- Правильный порядок импортов: стандартная библиотека → сторонние → локальные

### Docstrings

Все классы и функции должны иметь docstrings в формате Google style:

```python
def example_function(param1: str, param2: int) -> bool:
    """
    Краткое описание функции.
    
    Args:
        param1: Описание первого параметра
        param2: Описание второго параметра
        
    Returns:
        Описание возвращаемого значения
    """
    pass
```

### Type Hints

Используйте type hints где возможно:

```python
def get_client(phone: str) -> Client | None:
    """Получить клиента по телефону."""
    pass
```

## Инструменты разработки

### Установка

```bash
pip install -r requirements-dev.txt
```

### Проверка кода

```bash
# Проверка стиля кода
flake8 .

# Проверка типов
mypy .

# Форматирование кода
black .
isort .
```

Или используйте Makefile:

```bash
make lint    # Проверить код
make format  # Отформатировать код
make check   # Проверить без изменений
```

## Процесс разработки

1. Создайте ветку для ваших изменений
2. Внесите изменения, следуя стандартам кода
3. Проверьте код линтерами (`make lint`)
4. Убедитесь, что все тесты проходят
5. Создайте Pull Request с описанием изменений

## Структура коммитов

Используйте понятные сообщения коммитов:
- `feat: добавлена форма обратной связи`
- `fix: исправлена валидация телефона`
- `docs: обновлена документация`
- `refactor: оптимизирован код под PEP8`
